How to pronounce PostgreSQL

product creator-clarified

post gress Q L /ˈpoʊstɡrɛs kjuː ˈɛl/ mp3
or: post gress sequel /ˈpoʊstɡrɛs ˈsiːkwəl/

PostgreSQL is most commonly pronounced "post gress Q L" (/ˈpoʊstɡrɛs kjuː ˈɛl/). This pronunciation is documented by the project itself (see PostgreSQL FAQ).

FAQ documents "post-gres-Q-L" as official.

Alternate readings you might hear:

  • "post gress sequel" /ˈpoʊstɡrɛs ˈsiːkwəl/

Source: PostgreSQL FAQ

Pronouncing project and product names correctly avoids the small but persistent friction of being gently corrected during standups, conference Q&As, and team calls. Hearing the word a few times locks in the right reading better than reading IPA ever will. Pronounce is a community-maintained dictionary — every entry tagged with a confidence level and (where possible) a citable source.

Hear it from the command line

$ say-it PostgreSQL # primary × 3 + audible "or: …" for each alternate
$ say-it --alt PostgreSQL # focus on the first alternate
$ say-it --all PostgreSQL # primary AND every alternate, each repeated
$ say-it --solo PostgreSQL # primary only, no "or: …" tail
$ say-it --why PostgreSQL # print the dict entry with source URL

Install the CLI: git clone https://github.com/anzy-renlab-ai/pronounce.git && cd pronounce && ./install.sh
