## Submission etiquette

1. Read the list's CONTRIBUTING.md before opening a PR — most have a fixed alphabetical-within-section rule.
2. Make sure the project name links once and matches the casing of the GitHub repo / marketplace listing.
3. If the list has a static "checks" script, run it (`make` or `npm run check`) before pushing.
4. Don't submit to >2 lists per day from the same account — pattern-detected as spam.
5. After merge, **don't** post a "this list added me" tweet — looks tacky. Just let it sit.
