# Code of Conduct

## In short

Be kind. Be specific. Cite sources when you can. Don't fabricate them when you can't.

## In more words

This project is a community-maintained dictionary of how engineers actually pronounce things. Disagreements about pronunciation are normal — they're literally why the project exists. Disagreements should stay technical and good-faith.

**Expected:**

- Submit pronunciation entries with honest confidence levels. `creator-clarified` only when a real source documents a creator quote.
- Open issues that describe the problem, not the person.
- Welcome contributors who don't speak English natively — pronunciation perspectives from other languages are useful (Mandarin speakers caught the 故意 / GUI overlap, for example).
- Credit other contributors' work.

**Not expected:**

- Fabricated source URLs.
- Personal attacks, harassment, or "you're wrong, idiot"-style replies.
- Pronunciation-shaming. Many entries exist precisely because people have been corrected and felt bad about it.
- Spam or off-topic promotion.

## Enforcement

If you see something off, open an issue with the `code-of-conduct` label, or email the maintainer. Repeated violations result in blocks; the maintainer is the final judge.

This project follows the spirit of the [Contributor Covenant 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) — read that if you want the long version.
